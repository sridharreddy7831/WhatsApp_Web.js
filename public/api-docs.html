WhatsApp Multi-Session API v2

A highly concurrent, isolated whatsapp microservice designed to handle multiple connections simultaneously without overlap.

⚠️ Architecture Note:

For Single Users: You do NOT need a session ID! Simply call the root endpoints directly (e.g., /api/whatsapp/start). The system will automatically link you to a default bot session.

For Multi-Tenant Usage: You can isolate different WhatsApp numbers parallelly by adding a dynamic :sessionId parameter in the URL. For example, "John" can use /api/whatsapp/john/start and "Jane" can use /api/whatsapp/jane/start to safely isolate data sets simultaneously!
GET/POST /api/whatsapp/[:sessionId]/start
Spins up a headless Chromium container strictly isolated for the string provided in `:sessionId`. You must run this ONCE before any other commands mapping to this session ID.
Response Example
{
    "success": true,
    "message": "Session marketingTeam initializing..."
}
GET /api/whatsapp/[:sessionId]/status
Returns the connection state for the specific isolated session.
Response Example
{
    "sessionId": "marketingTeam",
    "ready": true,
    "status": "connected"
}
GET /api/whatsapp/[:sessionId]/qr
Retrieves the QR code base64 image (if the given session is awaiting to be scanned). Note that puppeteer boot takes up to 8 seconds so the QR may instantly not be available upon calling `/start`.
POST /api/whatsapp/[:sessionId]/pair
Alternative code pairing. Generates an 8-digit OTP string. (Client session must be initialized but NOT already connected).
Body Parameters
  • phone Phone to pair.Required
POST /api/whatsapp/[:sessionId]/send
Sends a text message using the specified connection pipeline.
Body Parameters
  • phone Target phone.Required
  • message Text.Required
POST /api/whatsapp/[:sessionId]/send-receipt
Sends a document/image utilizing Base64 binary.
Body Parameters
  • phone Target phone.Required
  • base64Data Binary base64.Required
  • mimetype E.g., "application/pdf"
POST /api/whatsapp/[:sessionId]/send-bulk
Background throttled mass-messaging utilizing the specified session worker.
Request Example
{
    "messages": [
        { "phone": "1234567890", "message": "Offer A" },
        { "phone": "0987654321", "message": "Offer B" }
    ]
}
GET /api/whatsapp/[:sessionId]/messages
Extracts recent chat history from a thread.
Usage Example
GET /api/whatsapp/john/messages?phone=9876543210&limit=15
POST /api/whatsapp/[:sessionId]/logout
Safely disconnects this specific session from WhatsApp, kills the puppeteer browser safely, and erases local session memory allowing it to be spun up freshly again next time.
POST /api/whatsapp/clear-all
GLOBAL EMERGENCY WIPE. Destroys every single isolated session concurrently running, completely wipes the local folder cache of WhatsApp Auth data, and releases all system RAM memory. Do not pass a session ID to this!
